Design rationale — /teach

Two ways to build a teaching skill

The reference teach skill in mattpocock/skills optimizes the artifact — it generates beautiful, reusable HTML lessons. This one optimizes the dialogue and the memory: which teaching method to use, on this learner, right now — and a stateful map of exactly where they are across months.

This skill — adaptive coaching loop

Governs the teaching, not the output

The pedagogy is the product. Files exist to carry it across sessions.

  • ◆Method chosen per move by a mastery ladder — instruct & worked examples for the brand-new, Socratic only once a scaffold exists, teach-back at the top. Operationalizes the expertise-reversal effect; it will not grill a novice.
  • ◆One breadth map = syllabus + live progress. You see the whole territory and a status on every area — "what's covered, what's left" in one file.
  • ◆Stateful cold resume. MAP + LOG let a fresh session pick up exactly where you stopped, weeks later.
  • ◆Recap mode. A distilled RECAP spine re-fires a topic's clean logic without re-reading the Q&A trail.
  • ◆Teach-back is the gate. Nothing is "done" until you can explain it — including where it breaks — unprompted.
  • ◆Grounded, not guessed. Breadth and every fact trace to RESOURCES; parametric guessing is banned.

Generates durable lesson artifacts

Knowledge / Skills / Wisdom; storage-strength over fluency-strength.

  • ▸Authors self-contained HTML lessons — one tightly-scoped concept each, Tufte typography, citations, cross-linked by anchors.
  • ▸ADR-style learning records (numbered 0001-…md) capture pivotal insights and drive what's taught next.
  • ▸Component reuse — shared stylesheets, widgets, simulators reused across lessons for consistency.
  • ▸ZPD targeting by reading prior records + mission, then picking the next relevant topic.
  • ▸Delegates "wisdom" to real-world communities — sends you to high-reputation forums to test skills authentically.
  • ✕No per-move method selector — invokes ZPD but gives no rule for instruct-vs-question by mastery; nothing stops Socratic-grilling a novice.
  • ✕No single breadth map with a status per area, and no explicit resume / recap phase — consolidation rides on the lesson + record artifacts.
The honest trade. The artifact approach wins where you want lasting, shareable teaching material — beautiful HTML lessons, interactive simulators, and a real-world "wisdom" loop this skill doesn't have. Choose it to produce lessons; choose this to be coached — adaptively, with a memory of where you are.
